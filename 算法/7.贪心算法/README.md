# 贪心算法

贪心算法（Greedy Algorithm）是一种在对问题求解时，总是做出在当前看来是最好的选择的算法。也就是说，不从整体最优上加以考虑，他所做出的是在某种意义上的局部最优解。

## 适用场景
贪心算法适用于满足以下两个条件的问题：

1. 贪心选择性质：通过局部最优选择能构建全局最优解
2. 最优子结构：问题的最优解包含其子问题的最优解

## 与动态规划的区别

- 最优解选择
    - 贪心算法并不总是能找到全局最优解，但它往往能在某种意义上得到近似最优解
    - 动态规划总是找到全局最优解，但其计算复杂度往往较高

-  子问题划分
    - 贪心算法通常是对问题的一个子问题进行求解，而动态规划则是对整个问题进行求解
    - 贪心算法通常是对每个子问题只做出一个选择，而动态规划则是对每个子问题都进行求解

## 算法优缺点
✔️ 优势：
- 时间复杂度通常较低（O(n log n)常见）
- 实现相对简单直观
- 适合实时系统和对时效性要求高的场景
	
❌ 局限性：
- 不能保证全局最优解（需严格证明有效性）
- 对问题特征有较高要求
- 调试困难（局部选择的影响具有传递性）
