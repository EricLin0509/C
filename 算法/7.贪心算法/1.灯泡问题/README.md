# 灯泡问题

## 问题描述

有 n 个灯泡，编号为 1 到 n

灯泡可以打开 (1) 或者关闭 (0)

当某个灯泡打开时，它右边的所有灯泡的状态会发生反转 (打开 -> 关闭，关闭 -> 打开)

问：至少需要打开多少个灯泡，才能使得所有灯泡都打开？

## 模拟

假设有三个灯泡，状态为 `[1, 0, 1]`

我们可以使用以下算法：

1. 如果当前灯泡是打开的，则跳过
2. 如果当前灯泡是关闭的，则打开它，并将它右边的所有灯泡关闭，然后增加计数器
3. 重复步骤 1 和 2，直到所有灯泡都打开

那么，对于 `[1, 0, 1]`，算法的执行过程如下

1. `[1, 0, 1]   count = 0`
2. `[1, 1, 0]   count = 1`
3. `[1, 1, 1]   count = 2`

[原算法](orig.c)

但这样的时间复杂度为 $O(n*n)$

## 优化——贪心算法

现在我们以 `[0, 1, 0, 1, 1, 0, 1, 1]` 为例

还是以使用之前的算法

1. `[0, 1, 0, 1, 1, 0, 1, 1]   count = 0`
2. `[1, 0, 1, 0, 0, 1, 0, 0]   count = 1`
3. `[1, 1, 0, 1, 1, 0, 1, 1]   count = 2`

此时，我们比较第一步和第三步的灯泡状态，可以发现后 5 位的灯泡状态相同，即 `[1, 1, 0, 1, 1]`

这就说明，如果打开灯泡两次，则右边所有灯泡的状态与原来的相同

这样，如果计数器的值为偶数，则不需要改变灯泡状态，否则需要改变

```c
int open_bulb(int *status, int n)
{
    int steps = 0;
    for (int i = 0; i < n; i++)
    {
        if (steps & 1 == 1) // 奇数步数，反转状态
            status[i] ^= 1;
        else // 偶数步数，保持状态
            status[i] &= 1;

        if (status[i] & 1 == 1) continue; // 灯泡已经打开，跳过
        else steps++; // 灯泡未打开，打开灯泡
    }

    return steps;
}
```

[优化算法](bulbs.c)

这样，时间复杂度从 $O(n*n)$ 降低到 $O(n)$

### 核心

从左到右遍历灯泡，每一步根据当前灯泡的实际状态 (考虑之前操作的影响) 决定是否打开：

如果当前灯泡是关闭的，则必须打开它，因为后续操作无法改变它，这是唯一能使它变亮的方法。这种局部最优决策最终得到全局最优解 (最少打开次数)
